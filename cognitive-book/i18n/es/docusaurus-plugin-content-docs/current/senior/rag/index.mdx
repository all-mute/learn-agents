## Глоссарий

В этом глоссарии собраны определения терминов, которые часто используются в документации.

<details>
  <summary>A</summary>

  **API (Application Programming Interface)**

  Набор правил и спецификаций, которым должно следовать программное обеспечение для взаимодействия друг с другом. API позволяет разработчикам использовать функциональность одной программы в другой, не вдаваясь в детали реализации.

  **Anthropic**

  Компания, занимающаяся исследованиями и разработками в области искусственного интеллекта, известная созданием Claude.

  **Anthropic Cookbook**

  Сборник рецептов, демонстрирующих, как использовать возможности Claude для достижения конкретных целей.
</details>

<details>
  <summary>C</summary>

  **Claude**

  Семейство больших языковых моделей (LLM), разработанных Anthropic.

  **Context Window (Контекстное окно)**

  Максимальное количество токенов, которое модель может учитывать при генерации ответа. Большее контекстное окно позволяет модели учитывать больше информации из истории разговора или документа, что приводит к более релевантным и последовательным ответам.

  **Конфиденциальность**

  Обязательство защищать информацию от несанкционированного доступа и раскрытия.
</details>

<details>
  <summary>D</summary>

  **Docusaurus**

  Генератор статических сайтов, используемый для создания документации.

</details>

<details>
  <summary>F</summary>

  **Fine-tuning (Тонкая настройка)**

  Процесс обучения предварительно обученной модели на новом, специфическом наборе данных для улучшения ее производительности в конкретной задаче.

  **Frontend (Фронтенд)**

  Пользовательский интерфейс приложения или веб-сайта, с которым взаимодействует пользователь.

  **Frontmatter (Фронтматтер)**

  Блок YAML или JSON в начале файла Markdown, содержащий метаданные о документе, такие как заголовок, описание и теги.
</details>

<details>
  <summary>I</summary>

  **Инференс**

  Процесс использования обученной модели для генерации прогнозов или ответов на новые входные данные.

</details>

<details>
  <summary>L</summary>

  **LLM (Large Language Model)**

  Большая языковая модель — это тип модели машинного обучения, обученной на огромном количестве текстовых данных для понимания и генерации человеческого языка.
</details>

<details>
  <summary>P</summary>

  **Prompt (Промпт)**

  Текстовый ввод, предоставляемый модели для генерации ответа. Промпты могут быть простыми вопросами или сложными инструкциями.

  **Prompt Engineering (Промпт-инжиниринг)**

  Процесс разработки и оптимизации промптов для получения желаемых результатов от языковой модели.

  **Production (Продакшн)**

  Реальная среда, в которой приложение или сервис доступен для пользователей.
</details>

<details>
  <summary>R</summary>

  **RAG (Retrieval-Augmented Generation)**

  Метод, при котором языковая модель дополняет свои знания, извлекая информацию из внешних источников, таких как база данных или интернет, во время генерации ответа.

  **Rate limiting (Ограничение скорости)**

  Механизм контроля количества запросов, которые пользователь или приложение может сделать к API в течение определенного периода времени. Это помогает предотвратить злоупотребления и поддерживать стабильность сервиса.
</details>

<details>
  <summary>T</summary>

  **Token (Токен)**

  Базовая единица текста, используемая моделями для обработки языка. Слова и знаки препинания обычно разбиваются на токены.
</details>
