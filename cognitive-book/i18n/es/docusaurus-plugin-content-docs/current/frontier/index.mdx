## Глоссарий

Здесь собран список терминов, которые часто используются в документации Humanlayer.

### A

#### Agent (Агент)

Программное обеспечение, которое взаимодействует с Humanlayer API для выполнения задач.

#### AI (Искусственный интеллект)

Общий термин для обозначения компьютерных систем, способных выполнять задачи, которые обычно требуют человеческого интеллекта.

#### AI Model (AI Модель)

Математическая модель, обученная на данных, которая может выполнять задачи, такие как классификация, регрессия, генерация текста и другие.

#### API

Интерфейс прикладного программирования. Набор правил и спецификаций, которым должно следовать программное обеспечение для взаимодействия друг с другом. Humanlayer предоставляет API для взаимодействия с платформой.

### B

#### Backend (Бэкенд)

Серверная часть приложения, которая обрабатывает запросы от клиентской части (фронтенда), работает с базами данных и выполняет другие задачи, необходимые для функционирования приложения.

### C

#### CLI

Интерфейс командной строки. Программа, которая позволяет пользователю взаимодействовать с операционной системой или приложением с помощью текстовых команд.

#### Cloud (Облако)

Сеть удаленных серверов, используемых для хранения, управления и обработки данных, а также для запуска приложений.

#### Codebase (Кодовая база)

Набор исходного кода, составляющего программный проект.

#### Container (Контейнер)

Стандартизированная единица программного обеспечения, которая упаковывает код и все его зависимости, чтобы приложение быстро и надежно работало из одной вычислительной среды в другую. Docker - популярная технология контейнеризации.

### D

#### Data Science (Наука о данных)

Междисциплинарная область, использующая научные методы, процессы, алгоритмы и системы для извлечения знаний и понимания из данных в различных формах.

#### Database (База данных)

Организованный набор данных, хранящихся и доступных в электронном виде из компьютерной системы.

#### Deployment (Развертывание)

Процесс выпуска программного обеспечения в продакшн.

#### Docker

Платформа для разработки, доставки и запуска приложений с использованием контейнеризации.

### F

#### Feature (Функциональность)

Отличительная черта, возможность или функция программного обеспечения.

#### Frontend (Фронтенд)

Клиентская часть приложения, с которой взаимодействует пользователь. Обычно это веб-сайт или мобильное приложение.

#### Frontmatter

Блок YAML или JSON в начале файла Markdown, содержащий метаданные о файле.

### H

#### Human-in-the-loop (Человек в цикле)

Подход к разработке AI-систем, при котором человек участвует в процессе обучения и принятия решений.

### L

#### LLM (Большая языковая модель)

Тип AI-модели, обученной на огромном количестве текстовых данных, которая может генерировать текст, переводить языки, писать различные виды контента и отвечать на вопросы информативным образом.

### M

#### Machine Learning (Машинное обучение)

Раздел AI, который позволяет компьютерным системам учиться на данных без явного программирования.

#### Model (Модель)

В контексте машинного обучения, это алгоритм, обученный на данных, который может выполнять определенные задачи, такие как классификация, регрессия или генерация текста.

### P

#### Pipeline (Конвейер)

Автоматизированный процесс, состоящий из нескольких этапов, для обработки данных или развертывания программного обеспечения.

#### Prompt

Текстовый запрос, который передается AI-модели для получения ответа.

#### Prompt Engineering (Инженерия промптов)

Процесс разработки и оптимизации промптов для получения наилучших результатов от AI-моделей.

#### Production (Продакшн)

Реальная среда, в которой используется программное обеспечение конечными пользователями.

### R

#### REST API

Архитектурный стиль для создания веб-сервисов. REST API использует HTTP-методы (GET, POST, PUT, DELETE) для доступа к ресурсам.

### S

#### SDK

Набор инструментов разработки программного обеспечения, который позволяет разработчикам создавать приложения для определенной платформы.

#### Server (Сервер)

Компьютер или программное обеспечение, которое предоставляет услуги другим компьютерам или программам (клиентам).

#### Software (Программное обеспечение)

Набор инструкций или программ, которые говорят компьютеру, что делать.

#### Stack (Стек)

Набор технологий, используемых для создания приложения. Например, LAMP (Linux, Apache, MySQL, PHP) или MERN (MongoDB, Express.js, React, Node.js).

### T

#### Token

В контексте AI, это наименьшая единица текста, которую обрабатывает модель. Например, слово или часть слова.
## Glosario

Aquí se recopila una lista de términos que se utilizan con frecuencia en la documentación de Humanlayer.

### A

#### Agent (Agente)

Software que interactúa con la API de Humanlayer para realizar tareas.

#### AI (Inteligencia Artificial)

Término general para sistemas informáticos capaces de realizar tareas que normalmente requieren inteligencia humana.

#### AI Model (Modelo de IA)

Modelo matemático entrenado con datos que puede realizar tareas como clasificación, regresión, generación de texto y otras.

#### API

Interfaz de programación de aplicaciones. Un conjunto de reglas y especificaciones que el software debe seguir para interactuar entre sí. Humanlayer proporciona una API para interactuar con la plataforma.

### B

#### Backend (Backend)

La parte del servidor de una aplicación que procesa las solicitudes de la parte del cliente (frontend), trabaja con bases de datos y realiza otras tareas necesarias para que la aplicación funcione.

### C

#### CLI

Interfaz de línea de comandos. Un programa que permite al usuario interactuar con el sistema operativo o la aplicación mediante comandos de texto.

#### Cloud (Nube)

Una red de servidores remotos que se utilizan para almacenar, administrar y procesar datos, así como para ejecutar aplicaciones.

#### Codebase (Base de código)

El conjunto de código fuente que compone un proyecto de software.

#### Container (Contenedor)

Una unidad estandarizada de software que empaqueta el código y todas sus dependencias para que la aplicación se ejecute de forma rápida y fiable de un entorno informático a otro. Docker es una tecnología de contenedores popular.

### D

#### Data Science (Ciencia de datos)

Un campo interdisciplinario que utiliza métodos científicos, procesos, algoritmos y sistemas para extraer conocimiento y comprensión de datos en diversas formas.

#### Database (Base de datos)

Un conjunto organizado de datos almacenados y accesibles electrónicamente desde un sistema informático.

#### Deployment (Despliegue)

El proceso de lanzamiento de software en producción.

#### Docker

Una plataforma para desarrollar, entregar y ejecutar aplicaciones utilizando la contenedorización.

### F

#### Feature (Funcionalidad)

Una característica distintiva, capacidad o función del software.

#### Frontend (Frontend)

La parte del cliente de una aplicación con la que interactúa el usuario. Normalmente, es un sitio web o una aplicación móvil.

#### Frontmatter

Un bloque YAML o JSON al principio de un archivo Markdown que contiene metadatos sobre el archivo.

### H

#### Human-in-the-loop (Humano en el bucle)

Un enfoque para desarrollar sistemas de IA en el que un humano participa en el proceso de aprendizaje y toma de decisiones.

### L

#### LLM (Modelo de lenguaje grande)

Un tipo de modelo de IA entrenado con una gran cantidad de datos de texto que puede generar texto, traducir idiomas, escribir varios tipos de contenido y responder preguntas de manera informativa.

### M

#### Machine Learning (Aprendizaje automático)

Una rama de la IA que permite a los sistemas informáticos aprender de los datos sin una programación explícita.

#### Model (Modelo)

En el contexto del aprendizaje automático, es un algoritmo entrenado con datos que puede realizar tareas específicas, como clasificación, regresión o generación de texto.

### P

#### Pipeline (Canalización)

Un proceso automatizado que consta de varias etapas para procesar datos o implementar software.

#### Prompt

Una solicitud de texto que se envía a un modelo de IA para obtener una respuesta.

#### Prompt Engineering (Ingeniería de prompts)

El proceso de desarrollar y optimizar prompts para obtener los mejores resultados de los modelos de IA.

#### Production (Producción)

El entorno real en el que el software es utilizado por los usuarios finales.

### R

#### REST API

Un estilo arquitectónico para crear servicios web. La API REST utiliza métodos HTTP (GET, POST, PUT, DELETE) para acceder a los recursos.

### S

#### SDK

Un conjunto de herramientas de desarrollo de software que permite a los desarrolladores crear aplicaciones para una plataforma específica.

#### Server (Servidor)

Una computadora o software que proporciona servicios a otras computadoras o programas (clientes).

#### Software (Software)

Un conjunto de instrucciones o programas que le dicen a una computadora qué hacer.

#### Stack (Pila)

Un conjunto de tecnologías utilizadas para crear una aplicación. Por ejemplo, LAMP (Linux, Apache, MySQL, PHP) o MERN (MongoDB, Express.js, React, Node.js).

### T

#### Token

En el contexto de la IA, es la unidad de texto más pequeña que procesa el modelo. Por ejemplo, una palabra o parte de una palabra.
