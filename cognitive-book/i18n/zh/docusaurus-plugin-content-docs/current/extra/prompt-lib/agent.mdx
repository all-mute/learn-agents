### 确定化（分解为子任务）

**系统提示词**

```text
你是一位项目管理专家。你的任务是将复杂的任务分解为更小、更易于管理的子任务。每个子任务都应明确定义并具有具体的结果。
```

**用户提示词**

```text
任务：开发并启动一个在线商店的新网站。
```

**LLM 回复**

```text
1. 市场和竞争对手调研。
2. 确定功能需求。
3. 创建设计原型。
4. 开发用户界面。
5. 设置后端。
6. 集成内容管理系统。
7. 测试网站。
8. 启动和监控网站运行。
```

### 规划、任务优先级排序

**系统提示词**

```text
你是一位时间管理顾问。你的任务是帮助用户根据任务的重要性和紧迫性，对任务列表进行优先级排序。
```

**用户提示词**

```text
任务：
1. 撰写报告。
2. 回复电子邮件。
3. 准备明天会议的演示文稿。
4. 进行市场调研。
```

**LLM 回复**

```text
1. 准备明天会议的演示文稿（高度紧急和重要）。
2. 撰写报告（高度重要，中等紧急）。
3. 回复电子邮件（中等紧急和重要）。
4. 进行市场调研（低紧急，但高度重要）。
```

### 总结并突出细节

**系统提示词**

```text
你是一位专业的编辑。你的任务是总结文本，同时保留关键细节和要求。
```

**用户提示词**

```text
{news_article}
```

**LLM 回复**

```text
文章要点：
- X 公司宣布发布新产品 Y。
- 该产品将于 12 月 1 日上市。
- 产品的主要功能包括 A、B 和 C。
- 预计该产品将通过创新技术改变市场。
```

### 决策（任务是否完成）

**系统提示词**

```text
你是一位质量控制专家。你的任务是根据提供的标准评估任务是否完成。
```

**用户提示词**

```text
任务：撰写一篇关于技术对教育影响的文章。
标准：
1. 至少 1000 字。
2. 包含来自现实生活中的例子。
3. 提及最新的研究。
```

**LLM 回复**

```text
任务已完成。文章包含 1050 字，包括来自现实生活中的例子，并引用了最新的研究。
```

### 委派给其他代理

**系统提示词**

```text
你是一位项目经理。你的任务是确定哪些任务可以委派给其他代理，并提出执行计划。
```

**用户提示词**

```text
项目：开发移动应用程序。
任务：
1. 界面设计。
2. 功能开发。
3. 测试。
4. 推广。
```

**LLM 回复**

```text
1. 界面设计 - 委派给设计师。
2. 功能开发 - 委派给开发团队。
3. 测试 - 委派给 QA 部门。
4. 推广 - 委派给营销团队。
```

### 批评任务执行，查找错误

**系统提示词**

```text
你是一位流程改进专家。你的任务是批判性地评估任务的执行情况，并找出可能的错误或遗漏。
```

**用户提示词**

```text
任务：准备季度销售报告。
```

**LLM 回复**

```text
报告已准备好，但缺少销售趋势分析。也没有考虑可能更全面地了解数据的季节性波动。
```

### 头脑风暴创意

**系统提示词**

```text
你是一位创意顾问。你的任务是为给定的主题生成新的想法。
```

**用户提示词**

```text
主题：用于财务管理移动应用程序的新功能。
```

**LLM 回复**

```text
1. 与社交网络集成以分享财务目标。
2. 基于人工智能的虚拟财务顾问。
3. 游戏化以实现财务目标。
4. 支持多币种账户。
5. 基于人工智能的支出分析，并提供节省建议。
