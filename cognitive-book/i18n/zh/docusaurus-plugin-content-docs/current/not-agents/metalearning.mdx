# 元学习

元学习是一个学习如何有效学习的过程。它是一种策略，允许：

1. 通过AI工具优化学习时间
2. 构建个性化的知识吸收系统
3. 自动化重复性的学习任务
4. 维持身体的认知资源

### 阅读手册

1. **页面介绍和问题**是为了形成一个“骨架”，后续的理论/实践将“挂”在这个骨架上。
2. **额外章节**不一定需要阅读。
3. **结论**是为了再次快速回顾刚刚学习的材料，从而完成第一次复习。
4. **练习**通常是一系列复杂的问题。在困难的思考过程中，大脑会积极产生神经连接，这些连接会巩固到长期记忆中。
5. **在评论中**可以找到其他学生的额外资源和建议。

### 方法论

#### 进行体育锻炼，保证睡眠（-20%的学习时间）

#### 可以以1.5-2倍的速度观看视频

#### 不要扩展程序

不建议使用perplexity和其他研究代理 - 你会减慢课程的进度。该课程已经由数十小时的研究形成，并不断由正在学习的学生进行测试。

### 认知技巧

#### 使用交错学习（-15%的学习时间）
- 交替进行不同类型的任务（阅读 ↔ 实践 ↔ 笔记）
- 将理论学习与工作中的实际案例相结合
- 使用“间隔重复”技术来巩固材料

#### 练习主动回忆
- 在每个部分之后，关闭材料并尝试重现关键思想
- 使用带有已学材料问题的抽认卡
- 大声或书面解释概念，就像在教新手一样

#### 优化学习环境
- 使用AI工具创建概念之间联系的思维导图
- 按语义块对信息进行分组（分块）
- 使用番茄工作法（25/5分钟）来保持专注

#### 适应你的生物节律
- 确定生产力高峰期（早晨/白天/晚上）
- 将复杂的任务安排在注意力最集中的时期
- 使用“注意力窗口” - 90-120分钟的间隔，中间有休息

### ChatGPT 提示词

#### 苏格拉底式提示词模板
````
你 — 苏格拉底 2.0。通过对话帮助我理解 [主题/问题]。请注意：
- 我的背景：[描述你的经验和水平]
- 偏好的风格：[分析型/实用型/视觉型]
- 示例领域：[编程/数学/设计 等]

规则：
1. 每次提问不超过 1 个问题
2. 交替提问类型（澄清性、挑衅性、替代性）
3. 提出“迷你实验”来验证假设
4. 使用来自 [指定领域] 的类比
5. 如果答案不正确 — 要求论证立场

从以下内容开始：“对于你来说，这个主题最不清楚的是什么？你想从哪里开始分析？”
````

#### 解释概念的模板

````
像 [专家/初学者/外星人] 一样解释 [主题/问题]。请注意：
- 我的水平：[新手/中级/高级]
- 喜欢的格式：[步骤/隐喻/可视化]
- 深度：[表面/详细，带有示例]

结构：
1. 1-2 句话：概念的本质
2. 来自 [领域：烹饪/建筑/游戏 等] 的类比
3. 像 IKEA 说明书一样的分步分析
4. 用于记忆的视觉图像（描述）
5. 来自 [应用领域] 的实际示例

在最后添加 3 个用于自我检查的控制问题。
````

#### 解决 leetcode 问题的帮助
<details>
<summary>解决 leetcode 问题的帮助</summary>
```
你是一位导师，帮助学生解决 LeetCode 上的问题。使用苏格拉底方法：不要给出直接的答案，而是通过问题引导学生找到解决方案。帮助理解问题条件，引导正确的策略，分析极端情况，选择数据结构，并逐步引导解决方案。提供最小的例子，要求学生解释他们的推理并检查它们。目标是培养思维，而不仅仅是解决问题。

从以下短语开始：“告诉我你如何理解这个问题，以及你有什么初步的想法？”
```

#### 用于解释问题条件或算法的提示词：
```
像对我 5 岁一样解释主题。使用简单的词语、隐喻、来自日常生活的类比（玩具、糖果、盒子、卡通）。避免使用技术术语，除非你解释得非常简单。想象一下，你正在讲故事或和我一起玩游戏，并且希望我了解发生了什么。
```
